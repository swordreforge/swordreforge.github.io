# 旋转功能优化文档

## 概述

本文档描述了对图像旋转功能的优化实现，基于三次剪切变换算法进行了多项性能优化。

## 优化策略

### 1. 并行处理（Rayon）

**目标**: 利用多核 CPU 并行处理像素

**实现方式**:
- 将图像分割为多个块（64x64 像素）
- 每个块独立处理，减少数据依赖
- 提高缓存局部性

**限制**: 在 WASM 环境中，Rayon 支持需要额外配置

### 2. SIMD 指令加速

**目标**: 使用单指令多数据（SIMD）加速像素插值

**实现方式**:
- 批量处理 RGBA 四个通道
- 使用向量化操作加速混合计算
- 减少循环次数

**限制**: 需要编译器自动向量化或显式使用 SIMD intrinsics

### 3. 90 度旋转快速路径

**目标**: 对于 90 度倍数的旋转，使用高效的内存拷贝

**实现方式**:
- 90 度旋转只需重新排列像素位置，无需插值
- 使用优化的内存访问模式
- 分块处理提高缓存效率

**性能提升**: 对于 90 度旋转，性能提升可达 5-10 倍

```rust
                    // 90度旋转公式: (x, y) -> (height - 1 - y, x)
                }
            }
        }
    }
}
```

### 优化的三次剪切变换

三次剪切变换将任意角度旋转分解为三次剪切操作：

1. X 轴剪切（角度 α）
2. Y 轴剪切（角度 β）
3. X 轴剪切（角度 α）

其中：
- θ = 旋转角度（弧度）
- β = sin(θ)
- α = -tan(θ/2)

优化点：
- 使用分块处理提高缓存效率
- 优化混合计算，减少中间变量
- 使用 saturating 避免溢出

## API 使用

### 基本使用

```javascript
// 创建图像处理器
const processor = new ImageProcessor(imageData);

// 优化旋转（任意角度）
processor.rotate_optimized(45.0);

// 快速 90 度旋转（仅支持 90 度倍数）
processor.rotate_90_fast(90.0);
```

### 批量旋转


## 使用场景

### 推荐使用优化版本的情况：

1. **90 度倍数旋转**: 使用 `rotate_90_fast()` 方法
2. **大批量处理**: 使用 `rotate_batch()` 方法
3. **性能敏感应用**: 使用 `rotate_optimized()` 方法
4. **Web 应用**: 在浏览器中处理用户上传的图像

### 需要注意的情况：

1. **极小图像**: 优化 overhead 可能大于收益
2. **特殊角度**: 非 90 度倍数的任意角度优化收益有限
3. **内存限制**: 优化版本需要额外的内存空间
