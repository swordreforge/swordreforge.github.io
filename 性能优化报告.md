性能优化清单

  ✅ 已完成的优化
   1. Canvas willReadFrequently 优化 - 为7个频繁操作的 Canvas 添加了
      willReadFrequently: true 选项
   2. 图片尺寸限制 - 限制最大图片尺寸为 8192x8192，防止内存溢出
   3. Canvas 上下文复用 - 避免重复创建 Canvas 上下文

  🔍 可优化的地方

  1. 减少重复的 to_base64 调用 (高优先级)
   - 位置：历史记录保存、预览模式保存

   - 问题：processor.to_base64() 会编码整个图像，对大图片很慢

   - 优化方案：
     - 使用 processor.get_bytes() 获取原始字节数据（更快）
     - 只在需要显示/下载时才转换为 base64
     - 考虑使用增量存储（只保存变化的部分）
     
     ⚠️ photon-rs 后端处理图像会使用base64编解码,注意

  2. 减少 updateCanvas 调用频率 (高优先级)
   - 位置：各种滤镜和操作后
   - 问题：updateCanvas() 被调用了 99 次，每次都会重绘整个画布
   - 优化方案：
     - 使用 requestAnimationFrame 批处理更新
     - 实现节流，避免短时间内多次调用
     - 只更新变化区域（脏矩形渲染）

  3. 优化历史记录存储 (中优先级)
   - 位置：saveToHistory() 函数
   - 问题：每次保存都使用 base64 字符串，占用大量内存
   - 优化方案：
     - 限制历史记录数量（已有 MAX_HISTORY）
     - 使用 OffscreenCanvas 存储历史
     - 考虑使用 Web Workers 处理历史记录

  4. 优化 ImageProcessor 实例创建 (中优先级)
   - 位置：多处 ImageProcessor.new_from_bytes()
   - 问题：每次都创建新的处理器实例，消耗内存
   - 优化方案：
     - 复用已有的 originalProcessor
     - 避免不必要的实例创建

  5. 优化离屏 Canvas (中优先级)
   - 位置：updateCanvas() 函数
   - 问题：每次更新都重新创建离屏 Canvas
   - 优化方案：
     - 复用离屏 Canvas，只更新尺寸变化时才重新创建
     - 使用对象池管理 Canvas 实例

  6. 优化 drawPolygonPreview (低优先级)
   - 位置：多边形抠图预览
   - 问题：每次点击都重绘整个画布
   - 优化方案：
     - 使用单独的 Canvas 层绘制预览
     - 避免每次都调用 updateCanvas() 重绘原图

  7. 优化事件监听器 (低优先级)
   - 位置：鼠标事件、窗口调整事件
   - 问题：事件监听器可能被重复添加
   - 优化方案：
     - 确保事件监听器只添加一次
     - 使用事件委托减少监听器数量

  8. 优化 base64 转换 (低优先级)
   - 位置：多处 base64 解码/编码
   - 问题：base64_to_bytes 和 atob 在大图片上很慢
   - 优化方案：
     - 缓存已解码的字节数组
     - 使用 Uint8Array 直接操作

  9. 优化 get_raw_pixels 调用 (低优先级)
   - 位置：自动抠图获取颜色
   - 问题：get_raw_pixels() 会复制所有像素数据
   - 优化方案：
     - 只读取需要的像素点
     - 使用 ImageData 的 data 属性直接访问

  10. 添加懒加载 (低优先级)
   - 位置：侧边栏工具
   - 问题：所有工具和参数面板都在页面加载时渲染
   - 优化方案：
     - 只在需要时才显示参数面板
     - 使用 CSS contain 属性优化渲染

✦ 📊 优先级总结
   - 高优先级：减少 to_base64 调用、减少 updateCanvas 频率
   - 中优先级：优化历史记录、ImageProcessor 实例、离屏 Canvas
   - 低优先级：预览绘制、事件监听、base64 转换、像素读取、懒加载
